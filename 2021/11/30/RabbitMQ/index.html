遵循AMQP协议，给予Erlang语言开发的

安装

1.新建/home/rabbitmq,把两个工具包上传

2.进入目录后，安装erlang

rpm -Uvh erlang-solutions-2.0-1.noarch.rpm

yum install -y erlang

erl -v 查看是否安装成功

yum install -y socat

3.解压rabbitmq

rpm -Uvh rabbitmq-server-3.8.13-1.el8.noarch.rpm

yum install rabbitmq-server -y

启动

systemctl start rabbitmq-server

查看状态

systemctl status rabbitmq-server

设置成开机自启动

systemctl enable rabbitmq-server

停止

systemctl stop rabbitmq-server

rabbitmq管理页面

1.默认是没有安装web端的客户端插件的，需要安装插件

rabbitmq-plugins enable rabbitmq_management

2.进入123.56.107.125:15672进入界面

ip：port（15672）

3.本机的话用guest guest

远程无法用guest访问

4.授权

docker pull rabbit mq:management

docker run 镜像id

测试发送后，发现消费者线程接受到了消息

思考？

绑定关系写在生产者还是消费者？

绑定并创建的配置文件，最好写在消费者里，因为队列是直接与消费者打交道的

通过注解还是配置文件声明？

最好是配置文件，因为配置文件更完善，更高级的比如死信队列等都是基于配置文件配置的

Linux安装redis

1.下载安装包 https://redis.io

2.上传到linux服务器，home目录下新建redis的文件夹里

tar -zxvf redis-6.2.4.tar.gz 解压

3.在解压后的文件夹中，安装基本环境命令

yum install gcc-c++

gcc -v检查是否安装成功

make

make install

4.进入redis默认安装路径 /usr/local/bin

mkdir myconfig

5.cp /home/jiang/redis-6.2.4/redis.conf myconfig

6.redis默认不是后台启动的，修改配置文件

vim myconfig/redis.conf

136行 daemonize no 改为yes

7.启动redis服务

/usr/local/bin目录下

redis-server myconfig/redis.conf

8.redis-cli -p 6379 以端口号6379启动客户端

改变ip以后，linux连接命令为 redis-cli -h 172.21.90.183 -p 6379 -a jiang

-h(ip地址，不是外网地址) -a(密码)

9.查看redis进程是否开启

新开一个窗口，ps -ef|grep redis

10.关闭redis服务

shutdown

exit

客户端和服务端进程就没有了

基础知识

1.默认16个数据库，使用第0个数据库，可以使用select切换数据库

select 2 –>切换到第三个数据库

2.查看所有的key

keys *

3.清空当前数据库/所有数据库

flushdb/flushall

4.redis是单线程，基于内存，cpu不是性能瓶颈，瓶颈是根据机器的内存，和网络带宽。

常用命令

exists key 判断key是否存在，存在1，不存在0

del key 移除当前key

move key index 移动除当前key到index库

expire key 10 设置10s中key过期

ttl key 查看剩余过期时间

set key value 设置key-value

get key 获取key对应的value

type key 查看key的类型

五大数据类型

String

set key value 设置key-value

get key 获取key对应的value

append key “hello” 给key追加hello字符串并返回value长度

strlen key 查看key对应的value的长度

incr key –key对应的value加1

decr key –key对应的value减1

incrby key [index] –key对应的value加[index]

decrby key [index] key对应的value减[index]

getrange key 0 2 –输出key对应value的前三个字符

setrange key 1 xx –替换key对应value，从下标为1开始替换

setex key 30 hello –创建key-value时就设置30s过期

setnx key value –不存在的时候才设置值，key存在则设置不成功

mset k1 v1 k2 v2 –一次性设置多个key-value

mget k1 k2 –一次性获取多个值

msetnx k1 v1 k3 v3 –原子操作，要么都成功，要么都失败，因为已经有k1了，所以k3也没有设置成功

事务

一次性，顺序性，排他性，没有隔离级别的概念

单条命令有原子性，事务不保证原子性

所有在命令中的事务，并没有直接被执行，只有发起执行命令才会执行Exec

·开启事务multi

·命令入队

·执行事务exec

·取消事务discard，事务中的命令都不会被执行

编译型异常（代码有问题，命令有错误），所有命令都不会执行

运行时异常（语法问题，eg：1/0），正确的命令会被执行

乐观锁

悲观锁：什么时候都会出问题，无论做什么都会加锁，影响性能，比如syncgicsed，用完释放锁

乐观锁：认为什么时候都不会出现问题，更新数据的时候判断下在此期间，是否有人修改过这个数据

获取version

更新的时候比较version

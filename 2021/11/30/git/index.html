常用命令

设置全局用户签名

注意：这里设置用户签名和将来登录 GitHub（或其他代码托管中心）的账号没有任 何关系。仅代表是本地客户端的用户签名

在路径C:\Users\Administrator.gitconfig文件内

查看全局用户签名

这只局部用户签名，局部签名用来与平台公钥签名用的邮箱相匹配

如果你的公司邮箱，github，gitee邮箱都不一致，需要拉取项目后，使用命令

设置成注册github或者gitee时的邮箱

初始化本地库

告诉git，获取这个目录的管理权

查看本地库状态

git status

第一行：在master分支

第二行：没有提交过任何

第三行：没有任何可以提交，暂存区为空

新建hello.txt文件再次

gitstatus

第三行：发现一个未被追踪（add）的文件，存在于工作区，使用git add追踪

添加暂存区

git add 文件名

这个警告是转换了行末换行符

用LF（linux） 替换 CRLF（windows）

再一次git status

git追踪到了这个hello.txt文件，目前在暂存区里

如果想删除暂存区里的hello.txt文件

git rm –cached hello.txt

仅删除暂存区，工作区（磁盘上的没有删除）

提交本地库

git commit -m “日志信息” 文件名

再一次查看状态

git status

查看版本信息

git reflog

git log更详细

修改

修改了hello.txt

再次git status

这时需要再次追踪一下

git add hello.txt

在提交

git commit -m “second commit” hello.txt

再次查看版本信息

git reflog

发现有两个版本，并且head指针指向第二个版本

版本穿梭

git reset –hard 版本号 不保存工作区内容

穿越回第一次提交的版本

查看hello.txt发现第一行没有1234

再次git reflog

发现指针指向了第一次提交的版本（第三行）

而第一行是日志，意思是指针指向了哪个版本号

版本穿梭其他命令

首次推送

情况1：已经有仓库，还没开始写代码，这样直接从idea，clone仓库，然后开始写代码

情况2：已经写了代码了，仓库才创建（空仓库什么都没有）

在项目的根目录下打开gitbush

git局部配置
git config use.name “git账号”
git config user.email “email”

初始化仓库
git init

然后正常写代码即可

#与远程仓库建立连接
git remote add origin “远程仓库地址”

#将本地仓库的文件推送到远程仓库
git push -u origin master

分支

查看分支

git branch -v

创建分支

git branch 分支名

切换分支

git checkout 分支名

合并分支

git merge 分支名

把指定分支合并到当前分支

正常合并

master分支

新建了一个a分支，内容没变之前也为

现在a分支改变内容为

add，commit以后

合并，注意要站在master合并a，就要先去master，发现可以合并

因为master分支没有改动，只改动了a分支是没问题的

异常合并

两个分支在同一个文件的同一个位置有两套完全不同的修改

就需要a改动，master分支也改动了，此时有异常

现在改动master内容

add，commit以后，切换a分支

add，commit以后，切换master分支

自动合并失败了，在hello.txt里有冲突

<<<和==之间是当前分支的代码

>>>和==之前是要合并的代码

就要手动删除不要的代码和符号

删除以后，还要add，commit（不能带文件名了）

在查看文件，发现合并成功了

私有github

yum -y install policycoreutils openssh-server openssh-clients postfix

添加 GitLab 镜像源并安装

curl -sS https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | bash

yum install -y gitlab-ce

修改地址

vim /etc/gitlab/gitlab.rb
将external_url改为对应的地址和端口

重载并启动

gitlab-ctl reconfigure

gitlab-ctl restart
